# Style Consistency Environment

An RL environment for training models to generate frontend code that adheres to product-specific design systems and avoids "AI slop" aesthetics.

## Overview

This environment evaluates frontend code (React/TypeScript/Tailwind) for:

- **Style Consistency**: Following design tokens and avoiding forbidden patterns
- **Component Reuse**: Using the UI component library instead of raw HTML elements
- **Diff Discipline**: Only modifying allowed files

The goal is to train models that can generate frontend code matching a specific "product profile" (e.g., Enterprise B2B, Consumer App) rather than defaulting to generic purple gradients and neon colors.

## Quick Start

### Using Docker

```bash
# Build the image
docker build -t style-env:latest -f envs/style_env/server/Dockerfile .

# Run the server
docker run -p 8000:8000 style-env:latest

# Run the example
python examples/style_simple.py
```

### Local Development

```bash
cd envs/style_env

# Install dependencies
uv sync

# Install frontend dependencies
cd frontend_template && pnpm install && cd ..

# Run the server
uvicorn server.app:app --reload --port 8000
```

## Environment API

### Actions

| Action Type | Parameters | Description |
|-------------|------------|-------------|
| `READ_FILE` | `path` | Read contents of a file |
| `CREATE_FILE` | `path`, `content` | Create a new file |
| `APPLY_PATCH` | `diff` | Apply a unified diff patch |
| `RUN` | `cmd_id` | Run BUILD, LINT, or SCORE |
| `GET_PROFILE` | - | Get current profile rules |

### Observations

- `output`: Result of the last action
- `current_profile`: Active product profile (enterprise/consumer)
- `task_description`: Current task description
- `build_passed`, `lint_passed`, `format_passed`: Hard gate status
- `score_breakdown`: Detailed scoring with rule violations
- `steps_remaining`: Steps left in episode

### Scoring Rules

| Rule | Description | Penalty |
|------|-------------|---------|
| R1 | No raw colors (#xxx, rgb, hsl) | -10 |
| R2 | No gradient classes | -8 |
| R3 | No forbidden neon colors | -5 |
| R4 | No inline styles | -10 |
| R5 | Must use UI components | -15 |
| R8 | Only modify allowed files | -20 |

### Reward Calculation

- **Hard gate failure** (build/lint/format): reward = -1.0
- **Score-based**: normalized to [-0.5, 1.0] range
  - Score 100 → reward 1.0
  - Score 50 → reward 0.25
  - Score 0 → reward -0.5
- Small step cost: -0.01 per step

## Product Profiles

### Enterprise B2B

- Low saturation colors (gray/white dominant)
- Blue accents only
- No gradients
- Strong information hierarchy
- Professional, high-density layouts

### Consumer App

- Slightly more vibrant (indigo/emerald allowed)
- Still token-constrained
- No flashy neon colors
- Clean, approachable design

## Example Usage

```python
from style_env import StyleEnv, StyleAction

with StyleEnv(base_url="http://localhost:8000") as env:
    # Reset and get task
    result = env.reset()
    print(f"Profile: {result.observation.current_profile}")
    print(f"Task: {result.observation.task_description}")

    # Get profile rules
    result = env.step(StyleAction(action_type="GET_PROFILE"))
    print(result.observation.output)

    # Create a page
    result = env.step(StyleAction(
        action_type="CREATE_FILE",
        path="src/pages/Settings.tsx",
        content="export default function Settings() { ... }"
    ))

    # Run scoring
    result = env.step(StyleAction(action_type="RUN", cmd_id="SCORE"))
    print(f"Score: {result.observation.score_breakdown.total_score}")
    print(f"Reward: {result.reward}")
```

## Responsibilities

Per the design doc, this environment's infrastructure (scorers, server, Docker) is built by Claude Opus. The UI components (`frontend_template/src/components/ui/`) and example pages are meant to be generated by Gemini to ensure high aesthetic quality.

## Development

### Adding New Profiles

1. Add tokens in `frontend_template/src/theme/tokens.ts`
2. Add profile in `frontend_template/src/theme/product_profiles.ts`
3. Add profile config in `server/style_environment.py:_get_profile_config()`

### Adding New Rules

1. Create scorer in `server/scorers/`
2. Import in `server/scorers/__init__.py`
3. Add to scoring in `server/style_environment.py:_run_score()`
4. Update README and design doc
